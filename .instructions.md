# DOMROV LMS - Development Instructions

You are helping develop features for **DOMROV**, an AI-powered code grading Learning Management System. Follow the established architecture patterns strictly.

## 🏗️ Architecture Overview

### Project Structure

```
apps/client_next/
├── app/(portal)/         # Authenticated pages (Next.js)
├── app/api/              # Internal API routes & services
├── ui/features/          # Feature-specific UI components
├── ui/hooks/             # Custom React hooks
├── ui/components/        # Shared components
├── lib/api/              # API client functions
├── lib/                  # Utilities & axiosInstance
├── types/                # TypeScript definitions
└── config/               # Configuration
```

## 📋 Feature Development Checklist

When creating ANY new feature, follow this exact order:

### 1. **DTOs & Type Definitions** (FIRST)

- File: `app/api/[resource]/dto.ts`
- Define `ApiResponse<T>`, resource DTO, request/response types
- Always include success flag and data wrapper in responses

### 2. **Service Layer** (SECOND)

- File: `app/api/[resource]/service.ts`
- Implement business logic using `createAuthorizedAxios`
- External API calls ONLY here
- Error handling with meaningful messages
- Return unwrapped data (not ApiResponse)

### 3. **API Route Handler** (THIRD)

- File: `app/api/[resource]/route.ts`
- Handle GET, POST, PATCH, DELETE requests
- Use action query params: `?action=list`, `?action=create`, etc.
- Return `{ success: true, data: result }`
- Token extraction: `request.headers.get('Authorization')?.replace('Bearer ', '')`
- Error handling with 500 status

### 4. **Client API Wrapper** (FOURTH)

- File: `lib/api/[resource].ts`
- ONLY client-side, calls internal Next.js routes (not external APIs)
- Export functions that call appropriate `apiClient` method
- Document with JSDoc

### 5. **API Client Export** (PARALLEL with #4)

- File: `lib/apiClient.ts`
- Add resource API section with methods
- Use `fetchAPI<ApiResponse<T>>()` wrapper
- Handle both GET and POST/PATCH/DELETE properly

### 6. **TypeScript Types**

- File: `types/[resource].ts`
- Define client-side interfaces matching DTOs
- Export from types barrel if needed

### 7. **Custom Hook**

- File: `ui/hooks/use[Feature].ts`
- Always export from `ui/hooks/index.ts`
- Manage state, loading, error, cleanup
- Use `mounted` flag to prevent memory leaks
- Return: `{ data, isLoading, error, actions... }`

### 8. **Feature Components**

- Folder: `ui/features/[feature]/components/`
- Cards: Display individual items (use `'use client'`)
- Client: Main component with hook + grid (use `'use client'`)
- Index: Export all components

### 9. **Page**

- File: `app/(portal)/[feature]/page.tsx`
- Import & render FeatureClient component
- Simple wrapper, no logic

## 🔑 Key Rules

### Response Format

```typescript
// ✅ CORRECT - Always this format
{ success: true, data: result }
{ success: false, error: "message" }  // with 400-500 status

// ❌ WRONG
{ result }
{ data: result }
```

### Data Flow

```
Component → useHook → lib/api/[resource].ts → lib/apiClient.ts
→ app/api/[resource]/route.ts → app/api/[resource]/service.ts
```

### Component Rules

- Mark **interactive** components with `'use client'`
- Server components for structure/layout only
- Import types from `@/types/`
- Import api from `@/lib/api/`
- Import hooks from `@/ui/hooks`

### Error Handling

- Service: try-catch, throw with message
- Route: catch, return `{ success: false, error }` with status
- API wrapper: throw Error for client to handle
- Hook: setState(error), handle in component
- Component: Show error message or EmptyState

### Loading States

```typescript
if (isLoading) return <div>Loading...</div>;
if (error) return <div className="text-red-600">Error: {error}</div>;
if (data.length === 0) return <EmptyState message="No items" icon="box" />;
```

## 📝 Naming Conventions

| Type       | Format           | Example            |
| ---------- | ---------------- | ------------------ |
| Folders    | kebab-case       | `my-feature`       |
| Files      | kebab-case       | `my-component.tsx` |
| Components | PascalCase       | `MyComponent`      |
| Functions  | camelCase        | `fetchUsers`       |
| Types      | PascalCase       | `UserDto`, `User`  |
| Constants  | UPPER_SNAKE_CASE | `API_TIMEOUT`      |
| Hooks      | camelCase        | `useNotifications` |

## 🎨 Styling Rules

- **Framework**: TailwindCSS (no CSS files for components)
- **Colors**: Use existing palette (blue, green, red, yellow, gray)
- **Spacing**: 4px units (p-4, mb-6, gap-3)
- **Typography**: Inter font (pre-configured)
- **Responsive**: Mobile-first (`md:`, `lg:` prefixes)
- **States**: hover, focus, disabled, active
- **Cards**: `border border-gray-200 rounded-lg shadow hover:shadow-lg`

## ✅ Before Marking Complete

- [ ] No `any` types (unless absolutely necessary)
- [ ] Error handling in place (try-catch + UI feedback)
- [ ] Loading states for all async operations
- [ ] Empty state for no data
- [ ] Null/undefined checks before accessing properties
- [ ] 'use client' on interactive components
- [ ] Hooks exported from ui/hooks/index.ts
- [ ] API client exported from apiClient.ts
- [ ] JSDoc comments on functions
- [ ] Responsive design (test mobile view)
- [ ] No console errors/warnings
- [ ] Consistent with existing code style

## 🚫 Common Mistakes (AVOID!)

1. Calling external APIs from components (use lib/api/ instead)
2. Forgetting `'use client'` on interactive components
3. Not unwrapping ApiResponse in service layer
4. Missing error boundaries/try-catch
5. No loading states
6. Using `any` types
7. Forgetting to export hooks from index.ts
8. Hardcoded values (use constants)
9. Missing null checks before .split(), .map(), etc.
10. Not handling nested API responses properly

## 🔄 Common Patterns

### Fetching Data in Hook

```typescript
useEffect(() => {
  let mounted = true;
  apiFunction()
    .then((data) => {
      if (mounted) setState(data);
    })
    .catch((err) => {
      if (mounted) setError(err.message);
    })
    .finally(() => {
      if (mounted) setIsLoading(false);
    });
  return () => {
    mounted = false;
  };
}, []);
```

### Safe Property Access

```typescript
// ✅ Safe
classItem.name && classItem.name.length > 0 ? classItem.name : "Default";

// ✅ Safe
item?.name?.split(" ") ?? [];
```

### Component with Hook

```typescript
'use client';
const { data, isLoading, error, action } = useFeature();
if (isLoading) return <Loading />;
if (error) return <Error msg={error} />;
return <div>{data.map(item => <Card key={item.id} item={item} />)}</div>;
```
